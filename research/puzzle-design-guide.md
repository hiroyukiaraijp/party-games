## 1. 全カテゴリ共通：良い問題の原則

### 1.1 「あーなるほど！」の設計（AHA体験）

良い謎解き問題の核心は **「内発的動機づけをもたらすシステム」** である。
参加者に以下の3つの感覚をもたらすことが目標：

| 感覚 | 説明 | 実装方法 |
|------|------|----------|
| **有能感** | 「自分の力で解けた」という達成感 | 知識ではなくひらめきで解ける設計 |
| **自己決定感** | 「自分で選んで取り組んでいる」感覚 | 強制感のない導入、複数アプローチの許容 |
| **他者受容感** | 「認められた」という感覚 | 正解時のフィードバック、共有の仕組み |

### 1.2 良い問題の5条件

1. **パッと見で解けそうに見える**: シンプルな見た目が解く意欲を高める。難解に見える謎は敬遠される
2. **答えが意味のある言葉**: 答えが「あさがお」「ありがとう」など意味ある言葉であること。無意味な文字列はNG
3. **知識不要・ひらめき重視**: 専門知識がなくても発想の転換で解ける
4. **一問一情報**: 一問に複数の情報や法則を詰め込まない
5. **答えを聞いて納得できる**: 「なるほど！」と思える論理的整合性がある

### 1.3 問題作成の基本3ステップ

```
ステップ1: 答えを決める（意味のある言葉・メッセージ）
ステップ2: 法則を考える（変換ルール・解き方の仕組み）
ステップ3: 答えと法則を組み合わせて問題に仕上げる
```

## 2. カテゴリA：謎解き問題（小謎パターン）

### 2.1 主要パターン一覧

#### パターン1: 五十音表参照型
- **仕組み**: 数字・座標・矢印で五十音表の特定マスを指定し、その文字を読ませる
- **バリエーション**: 行列番号指定、矢印方向指定、色分け指定
- **難易度調整**: 座標を直接与える（易）→ 座標自体を別の謎で導出（難）

**具体例1**: 「(8,3)(2,1)(5,5)(7,2)(10,1)」→ 五十音表で8行3列=お、2行1列=め、5行5列=で、7行2列=と、10行1列=う → **「おめでとう」**

> なぜ良い問題か: 五十音表という誰もが知る共通知識を使い、答えが祝福メッセージで意味がある。座標に気づけば確実に解ける。

#### パターン2: 漢字内カタカナ抽出型
- **仕組み**: 漢字の中に隠れているカタカナを読み取る
- **対象漢字例**: 矛（ム）、伝（テ）、化（ヒ）、仏（ム）、右（ナ）

**具体例2**: 「矛」「伝」「化」の中のカタカナを順に読む → ム・テ・ヒ → 解読して答えを導出

> なぜ良い問題か: 漢字をいつもと違う視点（部首ではなくカタカナとして）見るという発想転換が「あー！」を生む。

#### パターン3: 組み合わせ・マッチング型
- **仕組み**: ペアになる概念を線で結ぶと、交差点や経路上に答えが出現
- **素材**: ことわざの前後、対義語ペア、分割可能な言葉

**具体例3**: 「猫に→小判」「犬も歩けば→棒に当たる」「豚に→真珠」を線で結ぶと、線が交差する場所に文字が配置されていて答えが読める

> なぜ良い問題か: ことわざという共通知識を利用しつつ、「線を引く」という物理的操作が新鮮な体験になる。

#### パターン4: 文字消去型
- **仕組み**: 特定の文字・単語を除去すると残りが答えになる
- **バリエーション**: 直接指定、イラスト指定（「こけし」の絵→「こ」を消す）

**具体例4**: 「ありがとうおれい」から「おれい（お礼）」を消す → **「ありがとう」**

> なぜ良い問題か: 「消す」という逆転の発想が必要。答えが温かい言葉なので正解時に嬉しさが倍増。

#### パターン5: イラスト分数読み型
- **仕組み**: イラストをカタカナに変換し、分数指定で一部だけ読む
- **例**: イラスト4つ、各単語の1/3文字目を読む

**具体例5**: タマゴ(1/3=タ)、ベーコン(1/3=ベ)、モモ(1/2=モ)、キノコ(1/3=キ) → **「たべもの」は不成立、正確には指定位置に応じた読み**

> なぜ良い問題か: イラストの「名前」という変換と「分数」という数学的指定の組み合わせが二段階のひらめきを要求する。

#### パターン6: 言い換え・変換型
- **仕組み**: 言葉を別の表現に置き換える（数字、記号、英語等）
- **頻出変換**: 肉→2×9=18、26→アルファベットの数、50→五十音

**具体例6**: 「肉=?」→ にく → 2×9 → 18 → 答えは「18」

> なぜ良い問題か: 日本語の音と数字の二重性（に=2、く=9）を利用した日本語ならではのパズル。

#### パターン7: 穴埋め連想型
- **仕組み**: 関連性のある言葉を並べ、空欄の文字を抜き出すと答えになる
- **素材**: 時系列（一昨日→昨日→今日）、五十音順、序列

**具体例7**: 「□□①□→□□②→きょう」→ おとと①→き②う→きょう → 一昨日(おととい)→昨日(きのう)→今日 → ①=い、②=の → 答えは「いの（猪）」等

> なぜ良い問題か: 日常使う言葉の中から文字を抽出するという意外性がある。

### 2.2 複合パターン：魚の骨形式

複数の小謎の答えから各1文字を取り、矢印の先に並べて最終メッセージを完成させる構造。
リアル脱出ゲームの最も基本的な流れ。

```
謎① → 答え「さくら」 → 2文字目「く」
謎② → 答え「つみき」 → 1文字目「つ」  → 最終答え「くつ」
```

## 3. カテゴリB：なぞなぞ・言葉遊び

### 3.1 テクニック分類

#### テクニック1: ダジャレ型（言葉内隠語）
- **仕組み**: 答えの言葉の中に、問題のキーワードが隠れている
- **作り方**: 答えとなる言葉を決める → その中に別の言葉が含まれていないか探す

**具体例1**: 「パンはパンでも食べられないパンは？」→ **フライパン**
- 「パン」が含まれている + 食べられない → フライパン
- なぜ良いか: 子供でも「パン」に引っ張られ、キッチン用品に発想が飛ぶ瞬間が楽しい

**具体例2**: 「りっぱりっぱと言われるのに踏みつけられるものは？」→ **スリッパ**
- 「りっぱ」を含み、踏みつける用途 → スリッパ
- なぜ良いか: 「立派」と「スリッパ」の二重意味が笑いを生む

#### テクニック2: ぎなた読み型（区切り位置ずらし）
- **仕組み**: 言葉の切れ目を通常と違う場所で区切ると別の意味になる
- **名前の由来**: 「弁慶がなぎなたを持って」→「弁慶が、なぎなたを」or「弁慶がな、着なたを」

**具体例3**: 「ミネラルウォーターの輸出額が高い州は？」→ **ミズーリ州（水売り州）**
- 「水」+「売り」= ミズウリ → ミズーリ
- なぜ良いか: ミネラルウォーター=水、輸出額=売り、と変換に気づく二段階のひらめき

**具体例4**: 「マラソン大会で最下位になった五十音の文字は？」→ **り（リハビリ=「り」は「ビリ」）**
- 「り」+「は」+「ビリ」= リハビリ
- なぜ良いか: 「最下位=ビリ」と「リハビリ」の意外な接続。マラソン→怪我→リハビリの連想も美しい

#### テクニック3: なぞかけ型（ダブルミーニング）
- **仕組み**: 二つの無関係なものに共通する特徴を言葉の多義性で繋ぐ
- **形式**: 「○○とかけまして、××と解く。その心は□□」

**具体例5**: 「布団とかけまして、人生と解く。その心は、どちらも畳んだり敷いたりします」
- 布団: 物理的に畳む・敷く / 人生: 店を畳む・レールを敷く
- なぜ良いか: 全く異なる二つの概念が同じ動詞で繋がる意外性

#### テクニック4: 見立て型（比喩利用）
- **仕組み**: 物事を別のものに見立てて問題にする
- **古典例**: 「朝は四本足、昼は二本足、夕は三本足。これは何？」→ 人間（赤ちゃん→大人→杖をつく老人）

#### テクニック5: ひっかけ型（思い込み利用）
- **仕組み**: 問題文の表現が特定の思い込みを誘導し、実は別の解釈が正解
- **典型**: 数え方のトリック、同音異義語の利用

**具体例（追加）**: 「1000円の花瓶を持って走ったらいくらになった？」→ **1000円（値段は変わらない。「割れた」と思い込ませるひっかけ）**

### 3.2 なぞなぞの良問条件

| 条件 | 説明 |
|------|------|
| 答えが一つに定まる | 複数解釈可能だと納得感が薄れる |
| 言い換えの質が高い | 直接的すぎず、遠すぎず、絶妙な距離感 |
| 答えが身近な言葉 | 聞いて「知ってる！」となる言葉 |
| 問題文が自然 | 不自然な日本語だとヒントになりすぎる |
| 解説不要で理解可能 | 答えを聞けば仕組みがすぐわかる |

## 4. カテゴリC：暗号パズル

### 4.1 換字式暗号（文字を別の文字に置き換え）

#### 暗号1: シーザー暗号（五十音/アルファベットずらし）
- **仕組み**: 文字を一定数ずらす（例: 3文字後ろへ）
- **日本語版**: あいうえお表で「な→に→ぬ→ね→の」のようにずらす
- **難易度調整**: ずらす数をヒントで提示（易）/ 自分で発見（難）

**具体例1**: 暗号「QDCR」→ 各文字を3つ戻す → N,A,Z,O → **「NAZO」（謎）**
- なぜ良いか: 答え自体が「謎」で自己言及的。ずらし数に気づいた瞬間の快感がある

#### 暗号2: カナ入力キーボード置換
- **仕組み**: PCキーボードのかな入力配列でアルファベットキーを日本語に変換
- **例**: 「BQ5」→ こ・た・え → **「こたえ」**

**具体例2**: 暗号「F,S」をかな入力で読む → は・と → **「鳩」**
- なぜ良いか: キーボードという身近な道具の「別の面」に気づかせるAHA体験

#### 暗号3: フリック入力置換
- **仕組み**: スマートフォンのフリック入力配列（あかさたな...）の位置を数字と方向で表現
- **例**: 「1-中央」=あ、「1-左」=い、「2-中央」=か

**具体例3**: 「4中 2左 8右 3上」→ た・き・も・の → **「たきもの（焚き物）」**
- なぜ良いか: 毎日使っているスマホのフリック入力を暗号に使う現代的な発想

#### 暗号4: 上杉暗号（座標置換）
- **仕組み**: ひらがなを二桁の数字で表す。対応表の行列番号
- **歴史**: 戦国武将・上杉謙信由来とされる

**具体例4**: 対応表で「11=あ、12=い、13=う...」→ 暗号「31 22 43 11」→ さ・き・ほ・あ → 並べ替えて **「あさほき→朝吹き」等**
- なぜ良いか: 歴史的背景があり、対応表を発見する楽しさがある

#### 暗号5: Leet文字（形状類似置換）
- **仕組み**: 文字を形が似た別の文字・数字に置き換え
- **例**: S→5、I→1、E→3、A→4、O→0

**具体例5**: 「N4Z0T0K1」→ NAZOTOKI → **「謎解き」**
- なぜ良いか: デジタル文化を活用。形が似ている文字を見抜く視覚的な楽しさ

### 4.2 転置式暗号（文字の並び替え）

#### 暗号6: アナグラム
- **仕組み**: 文字を並び替えて別の言葉にする
- **ヒントの出し方**: 「左右交互に読め」「奇数番目→偶数番目の順で」

**具体例6**: 「たたいてはんえこ」→ ひらがなを並び替え → **「こたえはたんてい（答えは探偵）」**
- なぜ良いか: 「叩いて半エコ」という意味不明な文が「答えは探偵」に化ける驚き

#### 暗号7: 縦読み・斜め読み
- **仕組み**: 横書きの文章の各行頭（または特定位置）を縦に読むとメッセージが出現
- **応用**: 斜め読み、渦巻き読み、逆読み

**具体例7**:
```
「こんにちは元気ですか
 たまには連絡ください
 えんりょしないでね」
→ 各行頭を読む: こ・た・え → 「答え」
```
- なぜ良いか: 自然な文章に見えて、実はメッセージが隠されている二重構造

#### 暗号8: スキュタレー暗号
- **仕組み**: 一定幅で折り返して読む（古代ギリシャ式：棒に紙を巻きつけて読む）
- **デジタル版**: N文字ごとに改行して縦読み

**具体例8**: 「こくたりえあ」→ 3文字ごとに折り返し → 「こくた / りえあ」→ 縦読み: こり・くえ・たあ → **「こたえ＝くりあ（クリア）」**
- なぜ良いか: 物理的な「巻く」動作が暗号解読のメタファーになる

### 4.3 隠匿系（ステガノグラフィ）

#### 暗号9: 重ね合わせ型
- **仕組み**: 複数のシート・画像を重ねると隠されたメッセージが出現
- **デジタル版**: 半透明レイヤーの重ね合わせ、色フィルター

#### 暗号10: マイクロドット / 拡大型
- **仕組み**: 画像の一部に極小文字でメッセージが隠されている

### 4.4 暗号パズル作成の3原則

1. **手がかりを必ず示す**: ルール推測のためのヒントがゼロだと理不尽になる
2. **ルールはシンプルに**: 複雑な1ルールより、シンプルな2ルールの組み合わせ
3. **循環の明記**: 「ん→あ」「Z→A」に戻る場合は明示する

## 5. カテゴリD：脱出ゲーム問題設計

### 5.1 問題の階層構造

```
小謎（5〜10問）: 個人で解ける簡単な謎。パターン認識系
    ↓ 各答えから文字を抽出
中謎（1〜3問）: チーム協力が必要。複数の小謎の答えを組み合わせる
    ↓ 中謎の答えが大謎のヒントに
大謎（1問）: 発想の転換が必要。ストーリーと密接に関連
```

### 5.2 大謎の設計原則

大謎は「パターンが無限」であり、テンプレート化が最も難しい部分。
しかし以下の共通原則がある：

| 原則 | 説明 |
|------|------|
| **発想転換が核** | 時間をかけて解く謎ではなく、ひらめきの瞬間で解ける |
| **既出情報の再利用** | それまでの小謎・中謎・ストーリーの中にヒントがある |
| **ゲーム世界観との統合** | 登場人物の視点に立つ、物語の前提を疑う |
| **思い込みの破壊** | 「こうあるべき」という前提を覆す |

### 5.3 大謎で使われる発想転換テクニック

**具体例1: 形状の再解釈**
- 「I」を「1」に、「O」を「0」に見る → 文字と数字の境界を曖昧にする
- なぜ良いか: 目の前にあるものの「見方」を変えるだけという最小限のひらめき

**具体例2: メタ的利用**
- 問題用紙自体を折る・回転する・裏返すと答えが出現
- なぜ良いか: 「問題は紙の上にある」という思い込みを破壊

**具体例3: ストーリー前提の転覆**
- 「この部屋から脱出せよ」→ 実は部屋の外に出る必要はなく、別の条件を満たせばよい
- なぜ良いか: 物語世界の没入と論理パズルの融合

**具体例4: ゲームオーバー画面の利用**
- 偽のゲームオーバー画面に赤字で表示された数字が答えの一部
- なぜ良いか: 「失敗」だと思い込んでいた画面が実はヒントだった反転

**具体例5: スタッフの言葉に隠されたヒント**
- 開始前の説明・ルール説明の中に大謎の答えに繋がるキーワードが含まれている
- なぜ良いか: 注意力と記憶力が報われる仕組み

### 5.4 物理的ギミック（デジタル版への応用可能な概念）

| ギミック | 物理版 | デジタル応用 |
|---------|--------|-------------|
| 炙り出し | 火で温めて文字出現 | タップ/長押しで隠し要素出現 |
| UV塗料 | ブラックライトで可視化 | 色フィルター切り替え |
| 重ね合わせ | 透明シート重ねる | レイヤー/画面回転 |
| 鏡文字 | 鏡に映して読む | 画面反転操作 |
| 折り紙 | 紙を折って文字合成 | ドラッグで要素を重ねる |

## 6. カテゴリE：リアル脱出ゲーム（SCRAP型）パターン分析

### 6.1 小謎の頻出パターン（高頻度順）

SCRAP等のリアル脱出ゲームで10年以上・300回以上の参加から抽出されたパターン：

| # | パターン名 | 頻度 | 説明 |
|---|-----------|------|------|
| 1 | 五十音表参照 | 最高 | 座標・矢印で五十音表から文字を拾う |
| 2 | イラスト名変換 | 最高 | イラストの名称をひらがな化して操作 |
| 3 | 文字の穴埋め | 高 | 共通する文字を入れて言葉を完成 |
| 4 | クロスワード | 高 | 交差するマスから文字を抽出 |
| 5 | 漢字合成/分解 | 高 | 部首の組み合わせ、カタカナ抽出 |
| 6 | 色・形・数の法則 | 中 | 視覚的要素から法則を発見 |
| 7 | 英単語/アルファベット | 中 | A=1, B=2等の変換 |
| 8 | アナグラム | 中 | 文字並び替え |
| 9 | 迷路型 | 中 | 経路上の文字を読む |
| 10 | 虫食い算/魔方陣 | 低 | 数学的パズル |

### 6.2 頻出要素の組み合わせマトリクス

以下の要素が組み合わされて謎が構成される：

- **形**: 図形の形状、線の引き方
- **色**: 色分け、特定色だけ読む
- **順番**: 数字順、時系列、五十音順
- **位置（座標）**: 行列指定、上下左右
- **並び**: 法則的な配置
- **文字（頭文字含む）**: 各単語の頭文字を繋ぐ
- **数（文字数含む）**: 文字数が何かを示す
- **イラスト・画像**: 名称変換

### 6.3 SCRAP型イベントの全体構造

```
[導入] ストーリー説明・世界観構築（5-10分）
   ↓
[Phase 1] 小謎 5〜10問（15-20分）
   - 個人で解ける
   - パターン認識力が試される
   - 答えから1文字ずつ抽出
   ↓
[Phase 2] 中謎 1〜3問（10-15分）
   - 小謎の答えを組み合わせる
   - チーム協力が必要
   - 物理的操作（折る、重ねる等）
   ↓
[Phase 3] 大謎 1問（残り時間全て）
   - 発想の転換
   - ストーリーとの統合
   - 既出情報の再解釈
   ↓
[結末] 成功/失敗 → 解説
```

### 6.4 SCRAP型の良問具体例

**具体例1: 時計の法則**
- 問題: アナログ時計の絵が3つ。時刻は3:00, 6:00, 9:00。「次は？」
- 思い込み: 12:00と答えたくなる
- 正解: 時計の短針の位置がアルファベットの形（3時=L、6時=下向き、9時=左向き）
- なぜ良いか: 数列パターンという思い込みを破壊し、視覚的再解釈を要求

**具体例2: 地図と漢字の融合**
- 問題: 日本地図に矢印が書かれている
- 正解: 矢印が指す県名の一部を読む → メッセージ出現
- なぜ良いか: 地図という視覚情報と言語情報の融合

**具体例3: 折り紙メッセージ**
- 問題: 問題用紙に無意味な文字が散りばめられている
- 正解: 指示通りに紙を折ると、窓から見える文字だけで答えが読める
- なぜ良いか: 紙という物理的メディアの特性を最大限活用

**具体例4: セーブデータ型**
- 問題: 「おまえの記憶の中にある」と言われる
- 正解: ゲーム機のセーブ画面の進行度数値が答え
- なぜ良いか: ゲーム世界のメタ構造を利用した壁破り

**具体例5: 先頭文字ステガノグラフィ**
- 問題: 長い物語文が渡される
- 正解: 各段落の先頭文字を繋ぐとメッセージが出現
- なぜ良いか: 「普通に読んでいた文章」に実は別のメッセージが隠されていた驚き

## 7. 難易度調整の体系

### 7.1 難易度レベル定義

| レベル | 対象 | 特徴 | 例 |
|--------|------|------|-----|
| ★☆☆☆☆ | 初心者・子供 | 法則が提示済み、答えを当てはめるだけ | 五十音表＋座標が明示 |
| ★★☆☆☆ | 一般層 | 法則は提示されるが、適用にひらめきが必要 | 「数字=五十音」のヒントあり |
| ★★★☆☆ | 経験者 | 法則自体を発見する必要がある | 五十音表の存在自体がヒントなし |
| ★★★★☆ | 上級者 | 複数の法則を組み合わせる | 暗号解読→五十音変換→アナグラム |
| ★★★★★ | マニア | 発想の大転換が必要 | 問題用紙自体がヒント |

### 7.2 難易度調整テクニック

**易しくする方法:**
- 例示を入れる（「例: A=1の場合...」）
- 法則の一部を先に見せる
- 選択肢を提示する（4択化）
- ヒントを段階的に開示する

**難しくする方法:**
- 法則のヒントを減らす
- 複数ステップを必要にする
- 言い換えの距離を遠くする（「水」→「H2O」→「元素番号1と8」）
- メタ的要素を入れる（問題自体を疑う必要がある）

### 7.3 ヒントの出し方原則

```
ヒント1: 方向性を示す（「五十音表を使います」）
ヒント2: 具体的手順の一部（「数字は行と列を表しています」）
ヒント3: ほぼ答え（「(1,1)=あ です。同じように全て変換してください」）
```

**原則: ヒントは「考え方」を示し、「答え」は示さない**

## 8. アンチパターン集（悪い問題の特徴）

### 8.1 致命的な問題点

| アンチパターン | 説明 | 例 |
|---------------|------|-----|
| **理不尽暗号** | 手がかりゼロで暗号解読を要求 | ルール説明なしのオリジナル暗号 |
| **知識偏重** | 特定の専門知識がないと絶対に解けない | 「この元素記号は？」→ 化学知識必須 |
| **無意味回答** | 答えが意味のない文字列 | 答え「ぬれき」（何の意味もない） |
| **情報過多** | 一問に法則が3つ以上 | 暗号解読→アナグラム→五十音変換→漢字合成 |
| **曖昧な正解** | 複数の解釈が可能で正解が一つに定まらない | 「大きい動物は？」→ ゾウ? クジラ? |
| **滝川クリステル謎** | 説明不足で独自ルールが理解不可能 | 魔方陣のルール説明なし |
| **非常識前提** | 一般常識と異なる前提を暗黙的に要求 | 「5分時計」という存在しないもの |
| **作者の独りよがり** | 作者にしかわからない連想 | 「私の好きな色=?」→ 赤（知るか） |

### 8.2 よくある設計ミス

1. **テストプレイ不足**: 作者は答えを知っているので難易度を過小評価しがち
2. **専門用語の使用**: 「換字式」「転置」等の用語を説明なしに使う
3. **時間配分ミス**: 3分制限なのに解くのに10分かかる問題
4. **ヒントのないメタ要素**: 「紙を折れ」のヒントなしに紙を折らせる
5. **複数正解の放置**: 答えが複数あるのに1つしか正解にしない

### 8.3 チェックリスト（問題公開前に確認）

- [ ] 答えは意味のある日本語（または英語）か？
- [ ] 特別な知識なしで解けるか？
- [ ] 答えは一つに定まるか？
- [ ] ヒントなしでも解法の入り口がわかるか？
- [ ] 答えを聞いて「なるほど！」と思えるか？
- [ ] テストプレイで未経験者が解けたか？
- [ ] 問題文は自然な日本語か？
- [ ] 制限時間内に解ける難易度か？
- [ ] 一問に含まれる法則は2つ以下か？
- [ ] 不快な内容・差別的な表現はないか？

## 付録: パズル生成エージェントへの実装指針

### A. 問題生成フロー

```
1. カテゴリ選択（なぞなぞ/暗号/謎解き/複合）
2. 難易度レベル決定（★1〜5）
3. 答えとなる言葉を決定（意味のある日本語）
4. パターンテンプレート選択
5. 法則を適用して問題文を生成
6. バリデーション（アンチパターンチェック）
7. ヒント3段階を生成
8. テスト（答えから逆算して解けるか確認）
```

### B. 答えの選定基準

- 3〜6文字のひらがな/カタカナ語が最適
- 一般的に知られている言葉（小学校レベルの語彙）
- ポジティブな意味を持つ言葉が望ましい
- テーマやストーリーに関連する言葉

### C. パターン組み合わせの推奨

| 難易度 | 推奨パターン数 | 例 |
|--------|-------------|-----|
| ★1-2 | 単一パターン | 五十音表参照のみ |
| ★3 | 2パターン | イラスト変換 → 文字抽出 |
| ★4 | 2-3パターン | 暗号解読 → 五十音変換 → 頭文字読み |
| ★5 | メタ要素含む | 問題自体の構造を利用 |
