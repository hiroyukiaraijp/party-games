# 離脱防止（チャーン・プリベンション）

## 概要
ユーザー離脱の6カテゴリ、業界ベンチマーク、FTUE（初回体験）設計を含む離脱防止の包括的ルールを定める。

## 業界ベンチマーク

| 指標 | 全カテゴリ平均 | 教育/脳トレ | トップ事例（Duolingo） |
|------|-------------|-----------|---------------------|
| Day 1継続率 | 25-26% | 14-15% | - |
| Day 7継続率 | 11-13% | 5-7% | - |
| Day 30継続率 | 5.7-7% | 2-3% | 28%（無料ユーザー） |
| 月間離脱率 | - | - | 28%（2023年、47%から改善） |

- **目標**: Day 30で業界平均（2-3%）の5倍以上 = 10-15%を狙う

## 離脱の6カテゴリと対策

### 1. 技術的問題
- **症状**: クラッシュ、ロード遅延、バグ
- **対策**: 静的HTMLなのでサーバー起因の問題なし。JSエラーハンドリングの徹底、オフライン動作保証

### 2. 貧弱なUX / 認知的過負荷
- **症状**: 操作がわからない、情報量が多すぎる、UIが複雑
- **対策**:
  - FTUE（初回体験）でルールを「やりながら覚える」設計
  - 1画面の情報量を最小限に（認知負荷の節約）
  - ボタンは大きく（48px以上）、タップ領域は十分に

### 3. コンテンツ不足 / 退屈
- **症状**: 同じ問題の繰り返し、新しいゲームがない、飽きた
- **対策**:
  - DDA（動的難易度調整）で常に適切な挑戦レベルを維持
  - ゲームメカニクス自体の新奇性を定期的に追加
  - 問題バンクの拡充（静的JSONデータの継続追加）
  - 退屈は全離脱理由の31.6%を占める最大要因

### 4. 効果への疑念
- **症状**: 「本当に意味があるのか？」「ゲームが上手くなっただけでは？」
- **対策**:
  - 成長ダッシュボード（Insights）で客観的な進捗を可視化
  - 「このゲームは処理速度を鍛えます」と具体的なスキルを明示
  - 遠距離転移は約束しない（科学的誠実さ）

### 5. やらされ感（Streak Creepと燃え尽き）
- **症状**: ストリーク維持が義務化、楽しさの消失、途切れた瞬間に離脱
- **対策**:
  - ストリークフリーズ（救済措置）
  - ストリーク途切れ後の「リカバリーチャンス」提供
  - 「今日は短いセッションだけでOK」のマイクロセッション設計

### 6. 時間・金銭的コスト
- **症状**: 1セッションが長すぎる、課金圧が強い
- **対策**:
  - 1ゲーム2-5分で完結する設計（隙間時間にプレイ可能）
  - 完全無料（広告なし、課金なし）= 金銭的障壁ゼロ
  - 「もう1回」ボタンで手軽にリプレイ

## FTUE（First Time User Experience）設計

### Day 1の黄金律
- Day 1に3つ以上のレッスン（ゲーム）を完了したユーザー: Day 30定着率50%向上
- **目標**: 初回訪問で最低3ゲームをプレイさせる

### FTUE設計原則
1. **チュートリアル不要**: ルール説明は最小限。1文で理解できるゲームから始める
2. **即座に遊べる**: ランディングから1タップでゲーム開始
3. **最初のゲームは「正解を知っている」状態で**: 初回は練習ラウンドとして成功体験を保証
4. **有能感の即時提供**: 初回プレイ後に「素晴らしい！処理速度スコア: ○○ms」等のフィードバック
5. **次のゲームへの誘導**: 終了画面に「次はこのゲームも試してみよう」CTA

### FTUE画面フロー
```
[1] ランディング（ロゴ＋「遊んでみる」ボタン）
    ↓ 1タップ
[2] プレイヤー名入力（スキップ可能、デフォルト名あり）
    ↓ 1タップ
[3] ゲーム選択（最も簡単なゲームを「おすすめ」としてハイライト）
    ↓ 1タップ
[4] ゲーム開始（ルール説明は1文＋図解、3秒で理解可能）
    ↓ プレイ（2-3分）
[5] 結果画面（スコア＋有能感フィードバック＋「もう1回」or「別のゲーム」）
```

## 離脱兆候の検知と介入
```
兆候1: プレイ頻度の低下（週3→週1）
  → 介入: ストリーク復活チャンスの提供

兆候2: セッション時間の短縮
  → 介入: 新しいゲームの「お試し」を提案

兆候3: スコアの停滞（成長が見えない）
  → 介入: DDAの再調整、「新しいチャレンジ」の提案

兆候4: 7日間以上の不活動
  → 介入: 次回訪問時の「おかえりなさい」画面＋ストリーク復活
```
