# パズル・謎解き設計パターン

## 概要
良い問題の5条件、7つの主要パターン、致命的アンチパターンを定める。

## 良い問題の5条件

1. **パッと見で解けそう**: シンプルな見た目が解く意欲を高める。難解に見える謎は敬遠される
2. **答えが意味のある言葉**: 「ありがとう」「クリア」等の意味ある単語。無意味な文字列はNG
3. **知識不要・ひらめき重視**: 専門知識なしで発想の転換だけで解ける
4. **一問一法則**: 1つの問題に複数の法則を詰め込まない
5. **答えを聞いて納得できる**: 「なるほど!」と思える論理的整合性がある

## AHA体験の設計

### AHA体験の4特徴
1. **突然性**: 論理的推論の積み重ねではなく、突如として意識に跳躍
2. **即時解決**: ひらめいた瞬間に問題が完全に解けたことがわかる
3. **快の感情**: ドーパミン報酬系の爆発的活性化による強い快感
4. **直感的確信**: 「これが絶対に正しい」という疑いようのない確信

### AHA体験を生むメカニズム
- パズルに行き詰まっている間 → 緊張（フラストレーション）が蓄積
- 答えに気づいた瞬間 → 蓄積された緊張が一気に解放 → カタルシス
- fMRI研究: AHA体験時に海馬が爆発的活動 → 通常の2倍記憶に残る

## 7つの主要パターン

### パターン1: 五十音表参照型
- 座標・矢印で五十音表の特定マスを指定し文字を読ませる
- 例: 「(8,3)(2,1)(5,5)(7,2)(10,1)」→「おめでとう」
- 難易度調整: 座標直接（易）→ 座標を別の謎で導出（難）

### パターン2: 漢字分解・カタカナ抽出型
- 漢字の中に隠れたカタカナを抽出する
- 例: 「矛」→ ム、「伝」→ テ、「化」→ ヒ
- ゲシュタルト・シフト（文字を図形として再解釈）が核心

### パターン3: 組み合わせ・マッチング型
- ペアになる概念（ことわざの前後等）を線で結ぶと交差点に答え
- 例: 「猫に→小判」「犬も→棒に当たる」の線の交差

### パターン4: 文字消去型
- 特定の文字/単語を除去すると残りが答えになる
- 例: 「ありがとうおれい」から「おれい」を消す →「ありがとう」

### パターン5: イラスト分数読み型
- イラストをカタカナに変換し分数指定で一部だけ読む
- 二段階のひらめきを要求（名前の変換 + 分数の適用）

### パターン6: 言い換え・変換型
- 言葉を別の表現に置き換える（日本語の音と数字の二重性等）
- 例: 「肉=?」→ にく → 2×9 → 18

### パターン7: 穴埋め連想型
- 関連性のある言葉を並べ空欄文字を抜き出すと答えに
- 例: 「お□□い→き□う→きょう」→ 一昨日→昨日→今日

## 階層構造（小謎→中謎→大謎）

### 小謎（5-10問）
- 個人で解ける簡単なパターン認識
- 答えから1文字ずつ抽出して素材にする

### 中謎（1-3問）
- 小謎の答えを組み合わせる
- チーム協力が必要、情報統合の小パラダイムシフト

### 大謎（1問）
- 発想の大転換（水平思考）が必要
- ストーリーと密接に関連、既出情報の再利用

### 大謎の発想転換3パターン
1. **メタ的利用**: 問題用紙自体を折る/回転する/裏返すと答え出現
2. **形状・空間の再解釈**: 平面を立体に、2Dを3Dに見立てる
3. **ストーリー前提の転覆**: 信じていた目的や前提が実は違った

## 難易度調整の体系

| レベル | 特徴 | 例 |
|--------|------|-----|
| 1 | 法則が提示済み、当てはめるだけ | 五十音表＋座標が明示 |
| 2 | 法則は提示されるがひらめきが必要 | 「数字=五十音」のヒントあり |
| 3 | 法則自体を発見する | 五十音表の存在がヒントなし |
| 4 | 複数法則の組み合わせ | 暗号解読→五十音変換→アナグラム |
| 5 | 発想の大転換 | 問題用紙自体がヒント |

### ヒントの3段階
1. 方向性: 「五十音表を使います」
2. 具体的手順の一部: 「数字は行と列を表しています」
3. ほぼ答え: 「(1,1)=あ です。同様に全て変換してください」

**原則: ヒントは「考え方」を示し「答え」は示さない**

## 致命的アンチパターン

| アンチパターン | 説明 | 回避策 |
|---------------|------|--------|
| 理不尽暗号（エスパー謎） | 手がかりゼロ、作者の脳内にしかない論理 | 必ず手がかりを提示 |
| 知識偏重 | 専門知識必須 | 義務教育レベルの共通知識のみ |
| 無意味回答 | 答えが「ぬれき」等の無意味文字列 | 意味のある言葉を答えにする |
| 情報過多 | 一問に法則3つ以上 | 一問一法則の原則 |
| 曖昧な正解 | 複数解釈可能 | 答えが一つに定まる設計 |
| 作業感の強要 | ルールは簡単だが50文字の変換作業 | ひらめきの比率を高める |
| 作者の独りよがり | 作者にしかわからない連想 | テストプレイで検証 |

## 問題作成の3ステップ
1. 答えを決める（意味のある言葉・メッセージ）
2. 法則を考える（変換ルール・解き方の仕組み）
3. 答えと法則を組み合わせて問題に仕上げる
