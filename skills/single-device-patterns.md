# スマホ1台マルチプレイ設計パターン

## 概要
1台のスマートフォンで複数人が遊ぶための5つの設計パターンと、ブラインドスクリーンUI等の実装要件を定める。

## 5つの設計パターン

### 1. 画面共有型（Screen Sharing / Centerpiece）
- **メカニクス**: 端末をテーブル中央に置き、全員が同時に画面を見る
- **適したゲーム**: 処理速度系（ドブル型探索）、フラッシュ暗算、反応系
- **長所**: アナログボードゲームに近い視線集約、直接対話が促進される
- **短所**: 画面サイズの視認性限界、マルチタッチの競合、大人数で物理的に困難
- **実装ポイント**:
  - フォントサイズは最低24px、ボタンは最低48x48px
  - 色のコントラスト比4.5:1以上
  - 「口頭回答＋審判タップ」方式で物理的タッチ競合を回避

### 2. 回し読み型（Pass and Play）
- **メカニクス**: ターン終了後に物理的に端末を次のプレイヤーに手渡す
- **適したゲーム**: 深い思考が必要なもの（謎解き、ワーキングメモリ、言語系）
- **長所**: 情報の非対称性が構築しやすい、個人の深い思考時間が確保される
- **短所**: 待ち時間（ダウンタイム）が致命的、5人以上で体験劣化
- **実装ポイント**:
  - **ブラインドスクリーン必須**: 画面遷移間に「次のプレイヤーに渡してください。受け取ったらタップ」を挟む
  - 並行処理フェーズの設計（手元で考える課題を同時提供）
  - ターン所要時間の上限設定（タイマー表示）

### 3. 司会者型（Moderator / Host）
- **メカニクス**: 1人がデバイスを持ちゲームマスターとして進行、他者は物理空間で対話
- **適したゲーム**: 人狼系、NGワード、クイズ大会
- **長所**: 参加者の大半は画面不要で完全な対面対話が実現、低年齢/高齢者も参加可能
- **短所**: 司会者の認知負荷が高い、司会者自身がゲームの駆け引きに参加しにくい
- **実装ポイント**:
  - 司会者向けUIはシンプルな「次へ」「正解」「不正解」ボタン主体
  - 自動進行オプション（司会者不要モード）も用意
  - 大人数対応で最も有効なパターン（8-15人）

### 4. 同時秘密入力型（Simultaneous Secret Input）
- **メカニクス**: 各プレイヤーが他者に見られないよう秘密裏に入力を行う
- **適したゲーム**: ブラフ系、投票系、価値観スケール
- **長所**: 心理戦、社会的推論など高度なメタ認知メカニクスが単一デバイスで機能
- **短所**: 覗き見リスク、テンポの悪化
- **実装ポイント**:
  - 「タップ中のみ表示」パターン: `touchstart`で表示、`touchend`で非表示
  - 全員入力完了後に一斉開示する「投票→結果」フロー
  - 入力中は画面を手で覆うよう指示テキストを表示

### 5. タイマー協力型（Timer-based Cooperation）
- **メカニクス**: リアルタイム制限時間で全員が1台を操作/瞬時受け渡し
- **適したゲーム**: しりとり爆弾、記憶リレー、協力型パズル
- **長所**: 極度の時間プレッシャーによるパニックと直感的処理の強制、盛り上がり最大
- **短所**: デバイス落下/破損リスク、運動能力依存
- **実装ポイント**:
  - タイマーは視覚＋聴覚の両方でフィードバック（導火線アニメ＋効果音）
  - 残り時間が少なくなるほど背景色が変化（緑→黄→赤）
  - 物理的受け渡しの混乱を想定し、状態遷移をシンプルに保つ

## ブラインドスクリーンUI仕様
```
[構成要素]
- プレイヤー名の表示: 「○○さんの番です」
- 受け渡し指示: 「スマホを○○さんに渡してください」
- 確認ボタン: 「準備OK（タップして開始）」
- 覗き見防止: 背景を不透明な単色（#1F2937）で完全に覆う
- カウントダウン: タップ後3秒のカウントダウンで心理的準備時間を確保

[CSS要件]
- z-index: 9999（最前面表示保証）
- position: fixed; inset: 0;（画面全体を覆う）
- touch-action: none;（スクロール防止）
```

## パターン選択ガイド

| 人数 | 推奨パターン | 理由 |
|------|------------|------|
| 2人 | 画面共有型 or 回し読み型 | ダウンタイムが短く両方機能する |
| 3-5人 | 回し読み型 + タイマー協力型 | 適度な待ち時間、深い思考可能 |
| 6-10人 | 司会者型 + 同時秘密入力型 | 全員参加を保証 |
| 11-15人 | 司会者型（サブチーム分割） | 口頭回答＋投票で全員関与 |

## 組み合わせパターン
- 司会者型 × 回し読み型: 情報伝達のボトルネックを意図的に作り出しコミュニケーション自体をゲーム性に
- 画面共有型 × タイマー協力型: 中央の画面を囲んでの早押し＋制限時間
