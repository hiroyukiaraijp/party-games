# 競争 vs 協力の設計科学

## 概要
競争型・協力型メカニクスがプレイヤーの心理・学習効果・継続率に与える影響の科学データと、Party Games向けの設計原則を定める。

## 科学データ

### 競争型の効果
- **短期的覚醒**: 生理的覚醒水準を高め、エネルギー消費と努力量が急増
- **統計的学習の加速**: パターンの暗記や即時反応処理を強力に加速
- **注意力維持**: 他者の行動が自己の勝敗に直結するため、自分のターン以外でも注意力が維持される
- **リスク**: 敗北時のフラストレーション、自尊心の低下、長期的な心理的疲労
- **離脱率**: 個人プレイ（実質競争）グループの離脱率は64%

### 協力型の効果
- **内発的モチベーション**: 自己効力感と楽しさを有意に高め、プレイの継続を促進
- **離脱率**: ソーシャルプレイグループの離脱率はわずか15%（競争の1/4以下）
- **向社会的行動**: 協力ゲームを15分プレイしただけで「共有行動」が有意に増加
- **関係性**: SDTの「関係性」欲求を直接満たし、長期的な心理的ウェルビーイングに寄与
- **リスク**: フリーライダー現象（他者任せ）、自分のターン以外での注意力散漫

### 学習効果の比較
- 最終的な学習効果に決定的な統計的差異はない（どちらも個人学習より優位）
- 競争型: 暗記・反復系タスクで優位
- 協力型: 複雑な問題解決、長期エンゲージメントで優位

## 設計原則

### 原則1: デフォルトは競争型個人スコア
- Party Gamesの基本構造は「個人スコア型」
- 全ゲームに共通のスコアボード機能で個人間の比較を提供
- 短期的な覚醒と盛り上がりを生む基盤

### 原則2: 協力要素のレイヤリング
- 競争の上に協力的な体験を重ねる設計
  - チーム分割での対抗戦（チーム内は協力、チーム間は競争）
  - 「共同敗北」を許容する全員参加型ラウンド
  - 非対称な協力（得意な人が不得意な人をカバーできる構造）

### 原則3: ユーモアと定性評価へのシフト
- 数値的勝敗だけでなく「面白い回答」「独創的な発想」を投票で評価
- 競争の性質を「スキル」から「ユーモアの共有」へシフト
- 敗北のフラストレーションを無効化

### 原則4: 敗北の分散と排除の撤廃
- プレイヤー排除（途中脱落）は完全禁止
- 劣勢プレイヤーへのシステム的支援（ラバーバンディング）
  - 強力なヒントの提示
  - 得点倍率の変動
  - 逆転要素の付与

### 原則5: コミュニケーションの構造化
- 大人数では自由発言ではなく、システムが発言を「強制」する設計
- 各プレイヤーに固有の秘密情報・役割を与え「その人が発言しなければ進行しない」状況を作る

## ゲームタイプ別の推奨バランス

| ゲームタイプ | 推奨 | 理由 |
|------------|------|------|
| 処理速度系（タップバトル） | 競争90% / 協力10% | 短期覚醒が核心。短いラウンドで敗北ダメージが小さい |
| 記憶力系（神経衰弱等） | 競争70% / 協力30% | 他者がめくったカード情報の共有が協力的要素 |
| 言語系（しりとり等） | 競争50% / 協力50% | リレー形式で協力的、かつ個人スコアで競争 |
| 社会的知性系（人狼等） | 競争40% / 協力60% | 対話と推論が核心。心理的安全性が重要 |
| 謎解き系 | 競争20% / 協力80% | チーム全体での達成感がカタルシスを最大化 |

## 実装パターン: ラバーバンディング
```
仕組み:
- スコア差を監視（リアルタイム）
- 差が一定以上 → 下位プレイヤーに有利な調整を自動適用

調整例:
- 制限時間: 下位者+2秒、上位者-1秒
- ヒント: 下位者にのみ追加ヒントを表示
- ボーナス: 下位者の正解ポイントを1.5倍
- 逆転ラウンド: 最終ラウンドのみ3倍ポイント
```
