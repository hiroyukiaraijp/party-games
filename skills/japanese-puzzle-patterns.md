# 日本語特有のパズルパターン

## 概要
五十音暗号、漢字分解、フリック入力暗号、4種の文字体系活用、および回答正規化の実装ルールを定める。

## 五十音表を使った暗号パターン

### 座標指定型
- 五十音表は「母音(アイウエオ)×子音(アカサタナ...)」の二次元マトリクス
- 「あ＝(1,1)」「か＝(2,1)」のように行列番号で指定
- 全参加者が共有する強固なメンタルモデルを利用

### 方向ズラシ型（シーザー暗号の日本語版）
- 「あ」＋右矢印＝「い」、「あ」＋下矢印＝「か」
- 循環ルール: 「ん」の次は「あ」に戻る（明示必須）
- 難易度: ズラシ数をヒントで提示（易）→ 自力発見（難）

### 母音抽出型
- 「カラス」→ 母音を抽出 →「アアウ」
- 音の構造を視覚化し、隠れた法則に気づかせる

### 欠落補完型
- 「あいう○お」「かきく○こ」→ 「え」「け」を補完
- ゲシュタルト心理学の「閉合の法則」を直接利用

## 漢字を使ったパターン

### 部首の足し算
- 「木」＋「目」＝「相」
- 視覚的ブロックパズルとしての直感的わかりやすさ

### 引き算と残像
- 「海」−「毎」＝「氵（さんずい）」
- マイナス演算の「余り」を別の文字に見立てる

### カタカナ隠蔽（最頻出）
- 漢字の中に隠れたカタカナを抽出
- 例: 「矛」→ ム、「伝」→ テ、「化」→ ヒ、「仏」→ ム、「右」→ ナ
- 「兄」の中に「口」と「ル」、「印」に「エ」と「マ」
- 文字を図形として再解釈（ゲシュタルト・シフト）

### 四字熟語の共通項
- 「○肉○食」「一○一会」→ 共通の漢字を推理
- 熟語知識とパターン認識の組み合わせ

### 空間的配置
- 左に「日」右に「月」のイラスト → 合わせて「明」
- 視覚情報から漢字構造への変換プロセス

## 言葉遊びパターン

### ダジャレ型（言葉内隠語）
- 答えの中に問題のキーワードが隠れている
- 例: 「パンはパンでも食べられないパン」→「フライパン」
- 例: 「りっぱと言われるのに踏みつけられるもの」→「スリッパ」

### ぎなた読み型（区切りずらし）
- 言葉の切れ目を変えると別の意味に
- 例: 「ぱんつくった」→「パン作った」or「パンツ食った」
- 謎解きでは同音の再解釈トラップに応用

### なぞかけ型（ダブルミーニング）
- 2つの無関係なものに共通する特徴を言葉の多義性で繋ぐ
- 例: 「布団とかけまして人生と解く。どちらも畳んだり敷いたりします」

## 現代デジタル文化の暗号

### フリック入力暗号
- スマホのフリック入力配列を利用
- 「あ」の周囲に「い(左)う(上)え(右)お(下)」
- 例: 「4中 2左 8右 3上」→ た・き・も・の

### QWERTY配列暗号
- PCキーボードのかな入力配列でアルファベットを日本語に変換
- 例: 「BQ5」→ こ・た・え →「こたえ」

### Leet文字（形状類似置換）
- S→5、I→1、E→3、A→4、O→0
- 例: 「N4Z0T0K1」→ NAZOTOKI →「謎解き」

## 4種の文字体系の活用

### 文字体系間変換の設計
- ひらがな → カタカナ → 漢字 → ローマ字 の変換が武器
- 「りんご」「リンゴ」「林檎」「RINGO」: 表記で文字数・形状・法則が変化
- 「カタカナで入力せよ」という指示自体がヒント（後の問題で形状を利用する伏線）

### 同音異義語の活用
- 「コウエンにいけ」: 公園? 講演? 後援?
- 序盤は「公園」、終盤に情報追加で「講演」に反転

## 回答正規化の実装（必須）

### 問題: 正解なのにシステムが不正解と判定
- プレイヤーのフロー状態を最も破壊するUX問題

### 正規化処理（JavaScript実装）
```javascript
function normalizeAnswer(input) {
  let normalized = input;
  // 1. 前後のスペース削除
  normalized = normalized.trim();
  // 2. 全角→半角（英数字）
  normalized = normalized.replace(/[Ａ-Ｚａ-ｚ０-９]/g, s =>
    String.fromCharCode(s.charCodeAt(0) - 0xFEE0));
  // 3. カタカナ→ひらがな
  normalized = normalized.replace(/[\u30A1-\u30F6]/g, s =>
    String.fromCharCode(s.charCodeAt(0) - 0x60));
  // 4. 大文字→小文字
  normalized = normalized.toLowerCase();
  // 5. スペース除去（文字間）
  normalized = normalized.replace(/\s+/g, '');
  // 6. 長音符の統一（ー→ー）
  normalized = normalized.replace(/[―—–-]/g, 'ー');
  return normalized;
}
```

### 正規化ルール一覧
| 処理 | 入力例 | 正規化後 |
|------|--------|---------|
| 前後スペース削除 | 「 りんご 」 | 「りんご」 |
| 全角→半角（英数字） | 「ＡＢＣ１２３」 | 「abc123」 |
| カタカナ→ひらがな | 「リンゴ」 | 「りんご」 |
| 大文字→小文字 | 「RINGO」 | 「ringo」 |
| 文字間スペース除去 | 「り ん ご」 | 「りんご」 |
| 長音符統一 | 「コーヒー」 | 「こーひー」 |

### 正答判定の設計
- 正解データは正規化済みの形式で保持
- ユーザー入力も同じ正規化関数を通してから比較
- 複数正解がある場合は配列で保持: `["りんご", "apple", "林檎"]`
- プレースホルダーで入力形式を指示: 「ひらがなで入力」
