# ステルスアセスメント

## 概要
プレイヤーの没入感を阻害せずにゲーム内行動データから認知能力を推定する手法。ECD（証拠中心設計）の3モデル、ベイジアン推定、データ要件を定める。

## 基本概念
- **定義**: ゲームプレイ中のマイクロビヘイビア（微細な行動）から、テスト不安なしに認知能力を連続的に測定する技術
- **目的**: エンタメ体験を壊さず、プレイヤーの強み/弱みを自動的にプロファイリング
- **注意**: 我々のアプリでは医学的診断ではなく「ゲーム内パフォーマンスの可視化」に限定

## ECD（証拠中心設計）の3モデル

### 1. 能力モデル（Competency Model）
- 測定対象となる潜在変数の定義
- 我々の場合: 8つの認知ドメイン（ワーキングメモリ、処理速度、注意力、言語力、論理的思考、創造力、空間認識、社会的知性）
- 各ドメインは複数のサブスキルに分解可能

### 2. タスクモデル（Task Model）
- 潜在変数を引き出すためのゲーム内シチュエーション設計
- 例: N-back課題 → ワーキングメモリ
- 例: ストループ課題 → 注意力＋処理速度
- 1つのゲームが複数の潜在変数を同時に刺激

### 3. 証拠モデル（Evidence Model）
- プレイヤーの行動（観測変数）が潜在変数にどう結びつくかの統計モデル
- ベイジアンネットワーク: 各行動データが発生するたびに能力推定を動的更新

## 収集すべき行動データ（マイクロビヘイビア）

### 基本データポイント
| データ | 説明 | 対応する認知ドメイン |
|--------|------|-------------------|
| 反応時間（ms） | 刺激提示から初回操作まで | 処理速度 |
| 正答率 | 正解数 / 試行数 | 全般 |
| エラーの種類 | 誤タップ、未回答、タイムアウト | 注意力、衝動性 |
| 連続正解数 | ストリーク長 | ワーキングメモリ容量 |
| 回答パターンの変動 | アクション間の時間間隔のSD | 持続的注意 |
| リカバリー速度 | エラー後の次試行の反応時間変化 | 情動制御 |
| 戦略の変化 | 同一ゲーム内での行動パターン遷移 | メタ認知 |

### 社会的知性系の追加データ
| データ | 説明 |
|--------|------|
| 投票パターン | 誰に投票したかの一貫性 |
| ブラフ成功率 | 嘘を見破られなかった割合 |
| 協力行動頻度 | チーム利益を優先した選択の割合 |

## データ要件
- **推奨**: 15分のゲームプレイで約2,000データポイント
- **最低**: 1ゲーム（2-3分）で300-500データポイント
- **精度の目安**: 確認的因子分析で信頼性係数0.92-0.96（十分なデータ密度の場合）
- **収束的妥当性**: 従来の認知テストとの相関 r=0.50-0.72（複合スコア使用時）

## 簡易ベイジアン推定（静的JS環境向け）
```
概念:
- 各認知ドメインに初期値（事前確率）を設定
- ゲームプレイの結果（証拠）が得られるたびに事後確率を更新
- 尤度（このスコアが出る確率）× 事前確率 → 事後確率

実装（簡略版）:
- 各ドメインスコア = 直近N回の重み付き平均
- 重み: 直近ほど高い（指数減衰）
- 信頼度: データポイント数に応じて0-100%で表示
  - <10回: 「まだデータ不足」
  - 10-30回: 「傾向が見えてきました」
  - 30回+: 「安定したプロファイル」
```

## 可視化
- **レーダーチャート**: 8軸の認知プロファイル（CSS/SVGで描画）
- **成長グラフ**: 各ドメインの時系列推移（Canvas APIまたはCSS棒グラフ）
- **信頼度インジケータ**: データ量に応じたプログレスバー
- **比較表示**: 「前回比」の矢印表示

## 倫理的制約
- プロファイルは「ゲーム内パフォーマンス」であり「認知能力の診断」ではないと明示
- 医学的用語（ADHD、認知症等）は一切使用しない
- データはlocalStorageのみに保存（サーバー送信しない）
- ユーザーがいつでもデータを削除できる「リセット」機能を提供
